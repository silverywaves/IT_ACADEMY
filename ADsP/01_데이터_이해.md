## 데이터와 정보
1. 정의 & 특성 <br>


    - 추론과 추정의 근거를 이루는 사실
    - 관념적, 추상적 -> 기술적, 사실적
    - 라틴어 Dare(주다) 의 과거분사형(주어진 것)
    - 다른 객체와의 상호관계 속에서 가치를 가짐
  <br>
      
    - 존재적 특성 : 객관적 사실 - 데이터
    - 당위적 특성 : 추론, 예측, 전망, 추정을 위한 근거 - 정보
<br>
2. 유형 <br>


  - 정성적 데이터(Qualitative Data)
    - 비정형 데이터(주관식 응답, SNS에 올린 글 등 형태와 형식이 정해져있지 않은 데이터)
    - 주관적 내용 
    - 통계 분석이 어려움             
<br>

  - 정량적 데이터(Quantitative Date)
    - 정형 데이터(지역별 온도, 풍속, 강우량과 같이 수치로 표현되는 데이터)
    - 객관적 내용
    - 통계분석이 용이함(데이터 양이 크게 증가하더라도 DBMS에 저장, 검색, 분석하여 활용하기 용이함)
<br>

   - 정형 데이터
     - 정형화된 구조(고정된 필드) // 연산 가능
     - 수집, 처리 난이도 하
     - 엑셀, 스프레드시트, 관계형 데이터베이스(DBMS)
       
   - 반정형 데이터
     - 정형화된 구조(스키마, 메타데이터) // 연산 불가능
     - 수집 난이도 중, 보통 API 형태 => 데이터 처리기술(파싱) 필요
     - 데이터 존재(XML, JSON, HTML, 로그-웹로그&센서데이터)
   
   - 비정형 데이터
     - 정형화된 구조 X // 연산 불가능 // NoSQL 저장
     - 수집, 처리 난이도 상
     - 데이터 존재(소셜데이터-트위터&페이스북, 영상, 이미지, 음성, 텍스트)

   
<br>
3. 지식경영의 핵심 이슈


  - 암묵지
    - 개인에게 축적된 내면화된 지식 -> 조직의 지식으로 공통화
    - ex.김장김치 담그기, 자전거 타기 등         
<br>

  - 형식지
    - 언어, 기호, 숫자로 표출화된 지식 -> 개인의 지식으로 연결화
    - ex.교과서, 비디오, DB            
<br>

  - 암묵지와 형식지의 상호작용관계
    1. 공통화 : 암묵지를 타인에게 알려주기
    2. 표출화 : 암묵지를 책 등 형식지로 만들기
    3. 연결화 : 책 등에 자신이 아는 새로운 지식 추가하기
    4. 내면화 : 책 등을 보고 암묵적 지식 습득            
   <br>
4. DIKW 피라미드

   
    데이터, 정보, 지식을 통해 최종적으로 지혜를 얻어내는 과정을 계층구조로 표현
    - 데이터(Data)      : 가공하기 전의 순수한 수치나 기호
    - 정보(Information) : 데이터의 가공, 처리와 데이터간 연관관계 속에서 의미가 도출된 것
    - 지식(Knowledge)   : 상호 연결된 정보 패턴을 이해하여 예측한 결과물(의견)
    - 지혜(Wisdom)      : 지식의 축적과 아이디어가 결합된 창의적인 산물

## 데이터베이스의 정의와 특징
1. 연혁<br>


    - 1950년대 : 미국에서 군대의 군비상황을 집중 관리하기 위해 컴퓨터 도서관을 설립하면서 탄생
    - 1975년   : 우리나라에서 데이터베이스 이용이 이루어짐
    - 1980년   : 본격적인 데이터베이스 서비스 시대
    - 1980년대 중반 : 국내의 데이터베이스 관련 기술의 연구, 개발              
   <br>
   
2. 정의<br>


  - 1차 개념확대(정형데이터)
    - 데이터베이스의 법적 보호에 관한 지침(EU) : 체계적이거나 조직적으로 정리. 전자식 또는 기타 수단으로 개별적 접근 가능
    - 저작권법(국내) : 체계적으로 배열, 구성한 편집물. 개별적으로 접근 검색 가능               


  - 2차 개념확대(비형정데이터-빅데이터 출현)
    - 컴퓨터 용어사전(국내) : 동시에 복수의 적용 업무 지원
    - Wikipedia(국내) : 관련된 레코드의 집합. 소프트웨어로는 DBMS 의미
    - 데이터분석 전문가 가이드(국내) : 문자, 기호, 음성, 화상, 영상 등 체계적으로 수집, 축적            
   <br>
   
  3. 특징
    - 일반적 특징
      - 통합된 데이터  : 동일한 내용의 데이터 중복x
      - 저장된 데이터  : 컴퓨터가 접근할 수 있는 저장매체에 저장
      - 공용 데이터    : 여러 사용자가 공동으로 이용 -> 대용량화, 구조가 복잡
      - 변화되는 데이터 : 항상 변화하면서도 항상 현재의 정확한 데이터 유지 필요     <br>            

     - 정보의 축적 및 전달 측면에서의 특징
      - 기계 가독성 : 정보처리기기가 읽고 쓰기 가능
      - 검색 가독성 : 다양한 방법으로 정보 검색
      - 원격 조작성 : 정보통신망을 통해 원거리에서도 온라인 이용

## 데이터베이스의 활용
1. 기업내부 데이터베이스


  - 1980년대
      - OLTP(On-Line Transaction Processing)
          - 호스트 컴퓨터가 데이터베이스를 액세스하고, 바로 처리 결과를 돌려보내는 형태
          - 데이터 갱신 위주

      - OLAP(On-Line Analytical Processing)
          - 다양한 비즈니스 관점에서 쉽고 빠르게 다차원적인 데이터에 접근
            -> 의사 결정에 활용할 수 있는 정보를 얻을 수 있게 해주는 기술
          - 데이터 조회 위주
<br>

  - 2000년대 
      - CRM(Customer Relationship Management)
          - 고객 관계 관리
          - DB마케팅, 일대일마케팅, 관계마케팅 기반 <br>


2. 제조부문 데이터베이스 (CRM 포함)
      - ERP(Enterprise Resource Planning)
          - 전사적 자원 관리
          - 경영자원을 하나의 통합 시스템으로 재구축

      - BI(Business Intelligence)
          - 기업이 보유한 데이터를 정리, 분석해 의사결정에 활용
          - 질의(query), 보고(reporting), 온라인 분석처리(OLAP), 통계분석, 예측, 데이터마이닝의 결합

      - RTE(Real Time Enterprise)
          - 업무프로세스 중 발생하는 정보 신속 대응용
<br>
       
3. 유통부문 데이터베이스
   - SCM(Supply Chain Management)
     - 원재료 공급망 단계 최적화 -> 기업 고객에게 도달
    
    - KMS(Knowledge Management System)
      - 기업 보유 지식 통합 문제 해결 능력 향상
<br>
     
4. 금융부문 데이터베이스
   - EAI(Enterprise Application Integration)
     - 각종 서비스들 중앙시스템으로 연결하여 효율적인 서비스

## DBMS
- Data base Management System
    - 데이터베이스 관리, 응용 프로그램들의 데이터베이스 공유 사용 환경 제공 소프트웨어
    - 데이터베이스 구축 틀, 효율적인 데이터 검색, 저장 기능 제공
    - 오라클, 인포믹스, 액세스 등
<br>

- 종류
    - 관계경 DBMS
      - 컬럼(Column)과 로우(Row, 레코드 or 튜플)를 이루는 하나 이상의 테이블 또는 관계
      - 고유키(Primary Key)가 각 로우 식별
      - 테이블 : 하나의 엔티티 타입 대표
      - 로우 : 엔티티 종류의 인스턴스 대표
      - 컬럼 : 인스턴스 속성값 대표
    <br>
    
    - 객체지향 DBMS
      - 정보를 '객체' 형태로 표현
    <br>
    
    - 네트워크 DBMS
      - 레코드들이 노드로, 레코드들 사이의 관계가 간선으로 표현되는 그래프 기반
    <br>
    
    - 계층형 DMBS
      - 트리 구조 기반
<br>

- 데이터베이스 설계 절차
  - 요구사항 분석 -> 개념적 설계 -> 논리적 설계 -> 물리적 설계 -> 구현
<br>

- Relationship(관계)
  - 두 개의 엔티티 사이에 특별히 관리하고자 하는 직접적 관계(업무적 연관성)
    
  - 1:1 관계
    - 어느쪽 당사자 입장에서 상대를 보더라도 단 하나의 관계를 가짐
    - 현실에서 드문 케이스
    - 업무의 흐름에 따라 데이터가 설계된 형태

  - 1:m 관계
    - 한쪽은 1(one), 다른 한쪽은 m(many)
    - 자식은 부모를 하나만 가지고, 부모는 자식은 1명 이상 낳을 수 있음

  - m:n 관계
    - 서로가 서로를 1:N 관계로 봄
    - 한 회원은 여러 상품을 가질 수 있고, 한 상품도 여러 회원을 가질 수 있음
<br>

- 데이터 웨어하우스 & ETL
  - Data Warehouse(DW)
    - 조직 내 분산 운영되는 DBMS를 효율적으로 통합, 조정, 관리
    - 효율적 의사 결정을 위한 기초를 제공하는 실무적인 활용 방법론 제공
<br>

    - 특징
      - 주제지향성(Subject Oriented) : 주제 중심
      - 통합성(Integrated) : 데이터 통합
      - 시계열성(Time Variant) : 시간에 따른 변경 정보
      - 비휘발성(Non-Volatile) : 데이터 변경x. read only
<br>

    - ETL(Extract, Transform, Load)
      - DW 구축시 데이터를 운영 시스템에 추출하여 가공(변환, 정제) 후 DW에 적재하는 과정
<br>

- NoSQL
  - 분산 데이터베이스 기술 -> 확장성, 가용성
  - 비관계형 DBMS, 오픈소스
  - Not only SQL    <- SQL 계열 쿼리 언어 사용
  - key 와 value 형태로 자료 저장
  - 대용량 데이터 처리, 대규모의 수평적 확장성
  - MongoDB, Hbase, Redis, Cassandra 등

## SQL
- Structured Query Language
  - DB 접근을 위한 DB 하부 언어
  - 질의 기능 + 데이터 정의 + 조작
  - 테이블 단위로 연산 수행
  - 비교적 쉬운 사용 (영어 문장과 비슷한 구문)
<br>

- 분류
    - 데이터 정의어(DDL-Data Definition Language)
      - 테이블 관리 : 데이터 생성, 수정, 삭제 등 데이터 전체 골격 결정
      - CREATE, ALTER(속성변경), DROP, TRUNCATE
        <br>
     
    - 데이터 조작어(DML-Data Manipulation Language)
      - 테이블 저장 데이터 관리 : 레코드 조회, 수정, 삭제 등
      - SELECT, UPDATE, INSERT, DELETE
    <br>
    
    - 데이터 제어어(DCL-Data Control Language)
        - DB 접근, 객체에 권한 부여 : 데이터 보안, 무결성, 회복 정의
        - GRANT, REVOKE, COMMIT, ROLLBACK
<br>

- 집계함수
  - AVG(수치형) : 지정한 열의 평균값
  - COUNT(수치형, 문자형) : 테이블의 특정 조건이 맞는 것의 개수
  - SUM(수치형) : 지정한 열의 총합
  - STDDEV(수치형) : 지정한 열의 분산
  - MIN(수치형) : 지정한 열의 가장 작은 값
  - MAX(수치형) : 지정한 열의 가장 큰 값
  <br>

- 주요 구문
  - WHERE
    - SELECT, UPDATE, DELETE 문에서 특정 레코드에 대한 조건 설정
    - ROW 레벨 필터링 제공
   <br>
   
  - ORDER BY
    - 데이터를 지정된 컬럼으로 정렬
    - 기본값 ASC(오름차순)  // DESC(내림차순)
      <br>
      
  - GROUP BY
    - 데이터를 그룹별로 나눠 합계, 평균 등 연산
      <br>
      
  - HAVING
    - GROUP BY 를 통한 그룹별 연산 함수들 결과값에 조건식 설정시 사용
    - GROUP 레벨 필터링 제공
    - 독립적으로도 사용가능
    - 연산 함수들의 결과값은 직접 WHERE 절에서 조건식으로 사용 불가
