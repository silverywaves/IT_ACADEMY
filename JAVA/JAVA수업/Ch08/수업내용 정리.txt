멤버변수 ? 속성,필드 -> 객체의 개별정보 저장용
멤버메서드 ? 기능 -> 액션,동사형, 특정 역할 수행위한 코드블럭
메서드 기본구조 ? -> 
헤더(반환자료형 함수명 (파라미터..)  )+바디(처리 로직)

메서드 오버로딩 ? 함수 헤더가 바뀜
 => 다형성 : 클래스간의 상속관계를 가져 새로운 클래스 생성, 부모클래스의 메서드 사용 가능
 => 메서드 파라미터의 형태를 다양하게 둘 수 있도록 허용한 문법
 => 메서드 이름은 같게, 매개변수는 다양하게
 => 오버로딩할 메서드의 매개변수의 타입과 개수는 달라야 하며, 반환형은 같아야 함
   * 상속 <오버라이딩> : 부모클래스의 메서드를 추상화시켜 자식클래스에서 재정의하여 사용하는 것(부모의 기능을 자식이 고쳐서 사용) -> 함수 본체가 바뀜
 					=> 메서드명과 매개변수는 같아야 하며, 반환형도 같아야 함
 => 편하려고 사용하는 것
 
지역변수 ? {} 내에서 선언되는 변수 ,{}벗어나면 기본적으로 소멸 

생성자 ? 객체 생성시 1회 호출, 기본값 Setup 사용 메서드(함수) => 객체 생성시에 필요한 메모리공간 형성과 초기값 부여에 사용
생성자 오버로딩? 함수이기때문에 가능
 => 생성자 함수의 파라미터형을 다양하게 허용함으로써 외부로부터의 다양한 초기값을 허용
